# Contributing to KARO Booking

## Код

- Используйте TypeScript в строгом режиме. `any` запрещён.
- Пишите функциональные React-компоненты.
- Server Components применяются по умолчанию. Добавляйте `"use client"` только для браузерного состояния, эффектов или обработчиков событий.
- ESLint должен завершаться без ошибок перед каждым коммитом.
- Компоненты должны быть небольшими и выполнять одну задачу.
- Не храните бизнес-логику, тестовые данные и сложные преобразования внутри файлов страниц `src/app`.

## Архитектура

- Новая продуктовая функциональность размещается в `src/features/<feature>`.
- Общие UI-компоненты размещаются в `src/components/ui`.
- Компоненты административного интерфейса размещаются в `src/components/admin`.
- Маршруты и пункты меню определяются только в `src/config`.

## Проверка

Перед созданием pull request выполните:

```bash
npm run lint
npm run build
```
